# 零售门店一站式经营复盘 Skill

## 概述

本技能为零售门店运营打造单入口全闭环经营分析能力。基于用户上传的 Excel 原始经营数据，一键完成数据处理→指标计算→异常诊断→归因分析→报告生成。替代零售运营 80% 以上手工拼表、算指标、写复盘报告的重复工作。

## 触发条件

当用户执行以下任一操作时，使用本技能：

1. 上传零售经营相关的 Excel 文件（.xlsx / .xls），并提出分析/复盘/诊断需求
2. 明确要求"门店经营复盘"、"零售数据分析"、"业绩诊断"、"销售报表分析"、"门店对比"、"品类分析"、"经营报告"
3. 在零售经营场景下提出分析型问题（如"帮我看看这个月门店经营情况"、"分析下品类销售占比"、"对比各门店业绩"等）

## 核心执行流程（固定三段式闭环）

### 步骤一：数据清洗整合与指标计算

**目标**：将多张 Excel 表自动识别、清洗、合并，输出标准化财务报表和全套经营指标。

**执行方式**：运行 `scripts/data_processor.py`：

```bash
python scripts/data_processor.py <用户上传的Excel文件1> <用户上传的Excel文件2> ...
```

**脚本会自动完成**：
- 自动识别表类型（销售明细/品类销售/时段销售/外卖账单/会员订单/门店业绩）
- 字段智能匹配对齐（统一中英文列名）
- 数据去重、缺失补全、脏数据剔除
- 跨表合并归一
- 全套核心指标批量计算：销售额、销量、客单价、转化率、连带率、毛利率、促销让利占比、品类占比、时段占比、会员消费占比、业绩达成率等

**输出**：JSON 格式，包含 `数据汇总`（清洗后数据概览）和 `经营指标`（全套计算结果）。

**关键约束**：
- 若用户数据没有"客流量"列，按行业默认转化率 50% 反推，并在结果中标注"基于默认转化率反推，仅供参考"
- 若用户数据没有"成本"列，按行业默认 55% 成本率估算毛利率，并在结果中标注
- 确保指标计算准确率≥98%，跨表合并不产生重复数据

---

### 步骤二：经营异动智能归因诊断

**目标**：基于步骤一的指标结果，自动识别异常并完成根因分析。

**执行方式**：运行 `scripts/anomaly_detector.py`：

```bash
python scripts/anomaly_detector.py <步骤一输出的JSON文件路径>
```

**脚本会自动完成**：
- 七大异常场景扫描：整体业绩涨跌、客流波动、转化率波动、客单价波动、品类动销异常、毛利率下滑、节假日/高峰时段异常
- 基于阈值自动判定严重程度（P0 严重 / P1 关注 / P2 提示）
- 七维度归因：客流不足、转化率偏低、客单价下滑、促销到期、商品缺货、品类结构不合理、折扣促销过度
- 生成综合经营健康度评级（健康 / 一般 / 关注 / 预警）

**关键约束**：
- 所有归因必须贴合零售经营实际场景，不允许输出空泛或无关结论
- 如指标数据不足以支撑精确归因，标注"数据有限，以上归因基于合理推断"

**参考知识库**：
- `references/retail_knowledge.md`：行业基准值、归因维度与话术模板
- `references/metric_definitions.md`：所有指标计算公式与判定标准

---

### 步骤三：全自动生成标准化复盘报告

**目标**：整合数据、指标、异常、归因，组装结构化 HTML 报告。

**执行方式**：运行 `scripts/report_generator.py`：

```bash
python scripts/report_generator.py <步骤一输出的JSON> --diagnosis <步骤二输出的JSON> --type <日报|周报|月报> --mode <单店|多店> --output <输出文件路径.html>
```

**报告固定八项结构（强制输出，不可缺失）**：

1. **业绩总览** — 当期销售达成、同比/环比、目标完成率
2. **核心三要素拆解** — 客流量、转化率、客单价三维度分析及销售额拆解公式
3. **品类结构分析** — 各品类业绩贡献占比、TOP品类、短板品类
4. **门店盈利分析** — 整体毛利率、促销让利损耗、盈利基础
5. **经营风险提示** — 库存/业绩/促销/品类相关潜在风险
6. **会员经营复盘** — 会员消费基础数据、贡献占比
7. **核心问题汇总** — 结构化问题清单、经营健康度评级
8. **落地改善动作** — 逐条可执行整改方案，按 P0→P1→P2 优先级排列

**参数说明**：
- `--type`：日报/周报/月报，默认月报
- `--mode`：单店（单店独立复盘）或多店（多门店横向对比），默认单店
- `--output`：输出 HTML 文件路径

**报告样式**：内置响应式 CSS，支持桌面端和打印输出。品牌色使用紫色渐变系（#667eea → #764ba2）。

## 完整输出清单（全部产出）

每轮分析须向用户交付：

1. 标准化汇总数据表（步骤一 JSON 摘要）
2. 全套核心经营指标数据
3. 数据异常点位标注
4. 经营异常清单（P0/P1/P2 分级）
5. 业务归因解读
6. 整改优化方向
7. 数据可视化图表（品类占比条形图、指标卡片等，内嵌于 HTML 报告）
8. 完整标准化经营复盘报告（HTML 文件）

## 单文件快速模式（一键三步骤）

当用户期望快速得到结果时，将三个步骤串联执行：

```bash
# 1. 数据处理
python scripts/data_processor.py file1.xlsx file2.xlsx > result.json

# 2. 异常诊断
python scripts/anomaly_detector.py result.json > diagnosis.json

# 3. 生成报告
python scripts/report_generator.py result.json --diagnosis diagnosis.json --type 月报 --mode 单店 --output retail_report.html
```

生成后，向用户展示关键结论摘要，并指引其打开 HTML 报告查看完整内容。

## 重要提醒

1. **结论贴合实体零售**：所有分析结论和整改方案必须基于真实零售业务逻辑，杜绝空泛话术
2. **数据隐私**：仅在本地处理用户上传的 Excel 文件，不上传至任何外部服务
3. **容错处理**：若数据缺失关键字段（如客流量、成本），使用行业默认值估算并明确标注
4. **报告完整性**：HTML 报告必须包含全部八项固定结构，不可遗漏任何板块
5. **脚本依赖**：脚本依赖 `pandas`、`numpy`、`openpyxl`，首次运行前确保已安装：`pip install pandas numpy openpyxl`
