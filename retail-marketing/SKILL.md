# 零售营销材料生成

## 概述

本技能用于快速生成零售行业的高质量营销材料，包括公众号推文和朋友圈海报。内置五大零售场景的文案框架、四种海报视觉风格，支持品牌配置自动套用。推文输出可在手机端预览的 HTML，海报通过浏览器截图输出为 PNG 图片。

## 触发条件

以下任一情况满足时，使用本技能：

- 用户提到"推文""公众号""朋友圈""海报""促销""新品""节日营销"等零售营销关键词
- 用户要求生成零售行业的宣传内容
- 用户明确要求生成营销素材（而非普通文档）

**不适用于**：非零售行业内容、纯技术文档、非营销类写作。

## 核心能力

1. **公众号推文生成** — 适配微信手机端预览的 HTML 推文，含头部 Banner、正文区、CTA 区域、页脚
2. **朋友圈海报生成** — 四种视觉风格（渐变/极简/促销/产品叠层）的海报，支持竖版和方版尺寸，最终输出为 PNG 图片
3. **品牌配置管理** — 通过 JSON 配置文件管理品牌色、Logo、口号等信息，一次配置持续复用
4. **五场景覆盖** — 促销活动、新品上市、节日营销、会员忠诚度、清仓特卖

## 工作流程

### 第一步：确定场景与需求

从用户输入中提取关键信息：

| 需要确认的要素 | 示例 |
|---------------|------|
| 营销场景 | 促销 / 新品 / 节日 / 会员 / 清仓 |
| 材料类型 | 推文 / 海报 / 两者都要 |
| 行业细分 | 服装 / 美妆 / 家居 / 食品 / 3C 数码 / 其他 |
| 核心信息 | 活动主题、价格、时间、主打商品、CTA |
| 品牌配置 | 使用已有配置 / 本次临时指定 |
| 图片素材 | 用户提供 / 自动搜索 / 暂用占位图 |
| 小程序码 | 用户提供 / 上网搜索 / 占位符兜底 |

如信息不完整，优先向用户提问确认；如用户意图明确，直接推断并告知假设。

### 第二步：获取与处理图片素材

**重要**：推文和海报中必须包含真实图片，不得仅使用纯色占位块。图片获取与处理流程：

**2.1 确定所需图片清单**

根据场景和内容结构，列出需要的图片：

| 场景 | 推文图片需求 | 海报图片需求 |
|------|-------------|-------------|
| 新品上市 | 产品特写 2-3 张、场景展示 1-2 张、细节图 1-2 张 | 产品主图 1 张（可选背景图） |
| 促销活动 | Banner 促销图 1 张、爆品图 2-3 张 | 促销氛围背景图 1 张 |
| 节日营销 | 节日氛围图 2-3 张、礼盒/套装图 1-2 张 | 节日主题背景图 1 张 |
| 会员活动 | 会员权益图 1-2 张、礼品图 1-2 张 | 品牌形象图 1 张 |
| 清仓特卖 | 清仓商品图 2-3 张、价格标签图 1 张 | 促销氛围背景图 1 张 |

**2.2 优先询问用户提供**

生成内容前，向用户提问：

> "推文/海报需要配图，请提供以下图片：
> - 产品特写图（XX 张）
> - 场景/氛围图（XX 张）
> - 品牌小程序码（如有，将用于海报右下角）
> 
> 你可以直接发送图片文件，或者提供图片链接。如暂无图片，我将从网上搜索相关素材并处理后使用。"

**如果用户提供了图片**，记录文件路径或 URL，直接进入第 2.4 步处理。

**2.3 上网搜索图片（用户未提供时）**

1. 使用 **WebSearch** 搜索相关产品/场景图片，搜索关键词格式：`{品牌名} {产品名} 高清图片` 或 `{行业} {场景} 宣传图片`
2. 从搜索结果中提取可用的图片 URL（优先选择尺寸大、清晰度高的图片）
3. 使用 `curl` 或 PowerShell `Invoke-WebRequest` 下载图片到本地临时目录：
   ```
   临时路径：{工作目录}/outputs/.temp-images/img_{序号}.{ext}
   ```
4. 每张图片下载后验证文件是否完整（检查文件大小 > 5KB）

**2.3.1 小程序二维码获取（海报专用）**

小程序二维码放置在海报右下角，获取流程：

1. **先询问用户**：在第 2.2 步询问图片时，同步询问是否有小程序码图片可提供
2. **用户未提供则 WebSearch**：搜索 `"{品牌名称} 小程序码"` 或 `"{品牌名称} 小程序 二维码 高清"`，从结果中找方形二维码图片
3. **下载 & 处理**：下载后用 `process_image.py --width 120 --height 120 --quality 90 --output-base64` 处理
3. **默认二维码兜底**：搜索无结果或图片无效时，使用本 skill 内置的默认小程序二维码图片 `assets/images/default-qr-code.png`，经 `process_image.py` 处理为 base64 后赋值给 `{{QR_URL}}`，确保海报始终有可显示的二维码

**2.4 处理图片（必须执行，不可跳过）**

所有图片（用户提供或网上搜索）都必须经过以下处理才能嵌入 HTML：

**处理工具**：调用 `scripts/process_image.py`（位于本 skill 的 scripts 目录）

```bash
python {SKILL_DIR}/scripts/process_image.py \
  --input {原始图片路径} \
  --width {目标宽度} \
  --height {目标高度} \
  --quality {压缩质量} \
  --output-base64
```

**处理参数：**

| 用途 | 目标宽度 | 目标高度 | 压缩质量 | 说明 |
|------|---------|---------|---------|------|
| 推文 Banner 头图 | 750px | 400px（裁剪） | 85% | 适配手机 Retina 2x |
| 推文正文插图 | 750px | 自动（等比） | 85% | 宽度固定，高度自适应 |
| 推文产品卡片小图 | 180px | 180px（裁剪） | 80% | 正方形裁剪 |
| 海报产品主图 | 800px | 800px（裁剪） | 90% | 海报核心视觉（minimal/bold 风格） |
| 海报产品主图 (overlay) | 600px | 600px（裁剪） | 90% | overlay 风格居中大图展示 |
| 海报背景图 | 1080px | 1920px（裁剪） | 85% | 全幅背景 |

**处理规则：**
- **不得直接使用原始图片**：必须调整尺寸和压缩后再嵌入
- 裁剪模式：`cover`（保持比例，居中裁剪超出部分）
- 输出格式：统一为 JPEG（照片）或 PNG（需要透明时）
- `process_image.py --output-base64` 输出的 base64 字符串 **已包含 `data:image/jpeg;base64,` 完整前缀**，直接赋值给 `src` 属性即可，**切勿再次添加前缀**，避免出现 `data:image/jpeg;base64,data:image/jpeg;base64,...` 双前缀错误
- **重要：** `process_image.py` 的 process 日志输出到 stderr，stdout 仅输出纯净的 base64 字符串。脚本调用时直接捕获 stdout 即可获得可用数据，无需解析标记
- 推文单张图片 base64 不超过 200KB，超过则进一步压缩

**2.5 无可用图片时的降级方案**

如果搜索未找到合适图片或处理失败：
- 推文：使用渐变色图片占位符（`<div class="image-placeholder">`），但标注 `[待替换：请提供产品图]`
- 海报：保持纯品牌色方案，不强行加图片
- 明确告知用户哪些位置的图片需要后续替换

### 第三步：加载品牌配置与参考资料

1. 读取 `assets/brand-config.json` 获取品牌视觉配置
2. 如品牌配置尚未填写（仍为示例值），提示用户配置后可获得更佳效果，但使用默认值继续生成
3. 读取 `references/retail-copywriting.md` 获取对应场景的文案框架

### 第四步：生成推文（如需要）

使用 `assets/templates/article.html` 模板：

1. 读取模板文件内容
2. **变量名必须严格匹配**：模板中使用的变量名（如 `{{BG_COLOR}}`、`{{BORDER_COLOR}}`、`{{OFFICIAL_NAME}}`、`{{LOGO_URL}}`、`{{LOGO_ALT}}`）必须与生成脚本中的替换逻辑完全一致。常见易错点：
   - 模板用 `{{BG_COLOR}}`（不是 `{{BACKGROUND}}`）
   - 模板用 `{{BORDER_COLOR}}`（不是 `{{BORDER}}`）
   - 模板用 `{{OFFICIAL_NAME}}`（不是 `{{OFFICIAL_ACCOUNT}}`）
   - 模板用 `{{LOGO_URL}}` / `{{LOGO_ALT}}`（不是 `{{BRAND_LOGO_URL}}` / `{{BRAND_LOGO_ALT}}`）
   - 模板用 `{{QR_URL}}`（推文页脚可选）
3. 根据场景选择对应的文案结构（参考 `references/retail-copywriting.md`），生成 `{{CONTENT_BLOCKS}}` 内容
4. **⚠️ 模板和 CONTENT_BLOCKS 中均不得包含 HTML 注释（`<!-- ... -->`）**。原因：模板 `article.html` 顶部无 HTML 注释，CONTENT_BLOCKS 以 `<!-- Opening -->` 等开头时会提前闭合外部注释，导致后续内容泄漏到正文。已通过在 `article.html` 中彻底移除顶部注释修复。
5. 替换所有 `{{变量}}` 占位符，确保没有遗漏

**CONTENT_BLOCKS 生成规范：**

- 每个段落使用 `<p class="section-text">...</p>`
- 小标题使用 `<h2 class="section-heading">...</h2>`
- 产品卡片使用 `<div class="product-card">...</div>` 结构
- 列表使用 `<ul class="feature-list">...</ul>`
- 分隔线使用 `<div class="divider">···</div>`
- 统计数字使用 `<div class="stat-grid">...</div>`
- 高亮文字使用 `<span class="highlight">...</span>`
- 价格标签使用 `<span class="price-tag">¥XXX</span>`
- 引用块使用 `<div class="quote-block">...</div>`
- 金句/引用使用 `<div class="quote-block">...</div>`
- **正文插图**使用 `<div class="article-image"><img src="{{IMAGE_N}}" alt="..."><span class="image-caption">图注文字</span></div>`
- **全宽 Banner 图**使用 `<div class="article-image full-width"><img src="{{HERO_IMAGE}}" alt="..."></div>`
- **产品卡片图片**使用 `<div class="product-card-img" style="background-image:url({{IMAGE_N}})"></div>`
- 段落控制在 2-3 行（约 40-60 字），适配手机阅读
- 每 2-3 段插入一张图片，增加视觉节奏
- 所有图片 `{{IMAGE_N}}` 变量替换为处理后的 base64 字符串（`data:image/jpeg;base64,...`）

**推文长度建议：**

| 场景 | 建议段落数 |
|------|-----------|
| 促销活动 | 8-12 段 |
| 新品上市 | 6-10 段（更多图片） |
| 节日营销 | 5-8 段（情感为主） |
| 会员活动 | 7-10 段 |
| 清仓特卖 | 6-8 段（简明直接） |

4. 将替换后的 HTML 写入输出文件

**输出路径**：`{工作目录}/outputs/retail-article-{场景}-{日期}.html`

### 第五步：生成海报 HTML 中间文件

使用 `assets/templates/poster.html` 模板：

1. 读取模板文件内容
2. 根据场景选择视觉风格：

| 场景 | 推荐风格 | 风格标识 |
|------|---------|---------|
| 新品上市、品牌形象 | 产品叠层 | `overlay` |
| 节日营销、情感类 | 渐变风格 | `gradient` |
| 品牌形象、高端产品 | 极简风格 | `minimal` |
| 促销、折扣、清仓 | 大胆促销 | `bold` |

> **新品上市默认使用 `overlay` 风格**：产品大图居中展示，文字环绕分布在四周。尺寸为 1080×1440px（紧凑竖版），背景为纯白色搭配极淡的品牌色装饰圆（不遮挡内容）。布局：左上角品牌名 + 右上角序号 + 中上标题区 + 居中产品图（600×600）+ 左右标签卖点 + 底部标语+价格。需额外提供 `{{POSTER_EN_TITLE}}`、`{{POSTER_LEFT_TAGS}}`、`{{POSTER_RIGHT_TAG}}`、`{{POSTER_BOTTOM_SLOGAN}}`、`{{POSTER_NUMBER}}`、`{{POSTER_TAG}}`、`{{POSTER_PRICE_LABEL}}` 变量。

3. 根据场景和品牌配置生成海报内容，替换变量，并激活对应的风格区块
4. 将完整海报 HTML 写入中间文件（此文件为截图源，不直接交付）

**海报内容要素：**
- 主标题（4-10 字，简洁有力）
- 副标题（可选，补充说明）
- 核心卖点/价格（大字展示，建议使用 `{{POSTER_HIGHLIGHT}}` 占位）
- CTA（4-8 字简短号召语）
- 品牌 Logo 位置
- 二维码引导区
- **产品主图**（`{{POSTER_PRODUCT_IMAGE}}`，base64 编码，居中展示。overlay 风格下以 600×600 居中大图展示）
- **overlay 风格专用字段**：英文副标题 `{{POSTER_EN_TITLE}}`、左侧标签 `{{POSTER_LEFT_TAGS}}`、右侧标签 `{{POSTER_RIGHT_TAG}}`、底部标语 `{{POSTER_BOTTOM_SLOGAN}}`、序号 `{{POSTER_NUMBER}}`、副标题标签 `{{POSTER_TAG}}`、价格说明 `{{POSTER_PRICE_LABEL}}`、小程序二维码 `{{QR_URL}}`
- **其他风格字段**：品牌口号 `{{BRAND_SLOGAN}}`（gradient 底部）、高亮说明 `{{POSTER_HIGHLIGHT_LABEL}}`（minimal）、现价 `{{POSTER_PRICE}}` / 原价 `{{POSTER_PRICE_ORIGINAL}}`（bold）、二维码标签 `{{QR_LABEL}}`（bold）

**中间文件路径**：`{工作目录}/outputs/retail-poster-{场景}-{风格}-{日期}.html`

### 第五步（续）：截图导出为 PNG

1. 调用 **agent-browser** 技能，用浏览器打开上一步生成的海报 HTML 中间文件
2. **必须先设置视口尺寸**：使用 `agent-browser set viewport 1080 1440`（overlay 风格 1440px 高）或 `1080 1920`（其他风格），确保海报完整渲染在视口内
3. **全页截图**：使用 `agent-browser screenshot --full` 截取完整页面，避免 viewport 截断导致内容缺失
4. 将截图保存为 PNG 文件

**关键注意**：若不先设置 viewport 高度，浏览器默认视口仅 ~900px，1440px/1920px 高的海报会被截断，底部二维码、价格、标语等内容丢失。

**PNG 输出路径**：`{工作目录}/outputs/retail-poster-{场景}-{风格}-{日期}.png`

### 第六步：呈现结果

将生成的文件通过 present_files 工具呈现给用户：

- **推文**：直接呈现 HTML 文件，可在浏览器中手机模式预览（推荐 375px 宽度）
- **海报**：呈现 PNG 图片文件。HTML 中间文件仅作存档，不单独呈现

**如果同时生成推文和海报**，一并呈现，推文在前、海报在后。

**呈现后简要告知用户：**
- 推文可在预览面板中查看手机端效果
- 海报 PNG 可直接保存到手机发朋友圈。分辨率：默认竖版 1080×1920（gradient / minimal / bold），overlay 风格为 1080×1440；方版均为 1080×1080

## 特殊处理规则

### 价格展示
- 促销场景：同时显示现价和原价（划线），使用 `<span class="price-tag">` 和 `<span class="original-price">`
- 新品场景：只显示售价，突出"首发价"概念
- 会员场景：显示"会员价"和"非会员价"对比

### 颜色对比度
- 如果品牌主色亮度过高（HSL L > 80），Banner 文字自动切换为深色
- 确保所有文字区域满足 WCAG AA 对比度标准（至少 4.5:1）

### 节日特殊处理
- 春节：自动增加红色/金色元素，使用团圆、喜庆话术
- 情人节：自动使用粉色/红色调，增加心形装饰
- 双11/618：突出价格对比，增加紧迫感元素

### 合规检查
生成内容前，扫描文案中是否包含以下问题：
1. 广告法禁用词（"最""第一""国家级"等）→ 替换为合规表达
2. 无来源的数据宣称（如"销量第一"）→ 标注信息来源或移除
3. 虚构原价 → 使用"建议零售价"替代"原价"

### 图片处理规范

**图片获取优先级：**
1. 用户直接提供图片（最高优先级）
2. WebSearch 搜索下载（用户未提供时）
3. 渐变色占位图（兜底方案，需标注待替换）

**图片处理铁律（不可跳过）：**
- 所有外部图片必须经过 `scripts/process_image.py` 处理后才能使用
- 不得直接引用原始 URL 或直接嵌入原始文件
- 处理后输出 base64 字符串，写入 HTML 的 `src` 属性
- 推文内嵌图片总 base64 大小控制在 1.5MB 以内，超出则降低质量重压缩

**图片版权提示：**
- 网上搜索的图片仅供预览/草稿使用，质量和风格不可控
- 正式发布前必须替换为有版权的官方产品图片
- 生成后明确告知用户哪些图片来自网络搜索，建议替换
- 海报中的产品图建议由用户提供，搜索到的素材图片可能不匹配产品定位

### 小程序二维码处理规范

> 此节作为工作流第 2.3.1 节的补充说明，归纳了完整的处理逻辑。

**二维码获取优先级：**
1. **向用户获取**（最高优先级）— 第一步询问图片素材时，同步询问是否能提供品牌小程序码图片
2. **WebSearch 搜索**（用户未提供时）— 搜索 `"{品牌名称} 小程序码"` 或 `"{品牌名称} 小程序 二维码"`，从搜索结果中筛选方形二维码图片并下载
3. **默认二维码兜底** — 以上均无时，使用本 skill 内置的默认小程序二维码图片：`assets/images/default-qr-code.png`。将此图片经 `process_image.py` 处理为 base64 后赋值给 `{{QR_URL}}`，确保海报右下角始终显示可用的二维码，避免空白

**二维码处理规则：**
- 下载后经 `process_image.py` 处理：`--width 120 --height 120 --quality 90`（方形裁剪）
- `process_image.py` 输出的 base64 已含完整前缀，直接赋值给 `{{QR_URL}}` 即可
- 若用户未提供且搜索无结果，使用默认二维码：`{SKILL_DIR}/assets/images/default-qr-code.png`，同样经 `process_image.py` 处理后嵌入
- 默认二维码为通用小程序码样式，正式发布前建议替换为品牌真实小程序码

## 示例用法

**示例 1：促销推文**
> 用户："帮我写一篇夏装促销的公众号推文，全场 5 折起，活动只有 3 天"
> 执行：识别为促销场景 → 询问/搜索促销图片 → 下载并处理图片 → 加载品牌配置 → 读取促销文案框架 → 嵌入处理后图片生成 article.html → 呈现

**示例 2：节日海报**
> 用户："做一个情人节的海报，情侣套装 ¥520"
> 执行：识别为节日场景 → 选择渐变风格 → 询问/搜索产品图片 → 下载并处理为海报尺寸 → 加载品牌配置 → 生成 poster HTML → 浏览器截图 → 输出 PNG 图片 → 呈现

**示例 3：推文+海报组合**
> 用户："三八节快到了，帮我做推文和海报，主题是'悦己'"
> 执行：识别为节日场景 → 询问/搜索节日氛围图片 → 下载并分别处理（推文尺寸 + 海报尺寸）→ 加载品牌配置 → 先推文（HTML）后海报（HTML → 截图 → PNG）→ 一并呈现
